## 1. ビジネス要件

### 1-1. To-Be業務プロセス概要

AlphaCameraControllerは、カラーアライメント測定システム（CAS）の一部として機能するカメラ制御システムです。α6400（ILCE-6400）を制御し、U/F調整及び目地補正用の画像撮影やカメラのパラメータ設定を行うバックグラウンドサービスとして、カメラコントロールライブラリ（CameraControl）と連携して動作します。

本システムは以下の機能を提供します：
- カメラとの接続管理
- 撮影パラメータ（F値、シャッター速度、ISO感度、ホワイトバランス）の設定・制御
- フォーカスモードの制御
- タスクトレイからのアプリケーション制御
- 撮影制御（ライブビュー、JPG、RAW）
- CASメインシステムとのファイルベース制御通信（タイマーポーリング方式）
- CASによるAlphaCameraControllerプロセスの自動再起動（異常終了時の自己復旧）

---

### 1-2. 業務内容、業務特性（ルール、制約）

| 業務名 | 業務内容 | ルール・制約 |
|--------|----------|-------------|
| カメラ接続管理 | カメラとの接続確立・切断管理 | アプリケーション起動時に自動接続、カメラ名は設定ファイルで指定 |
| 撮影パラメータ管理 | F値、シャッター速度、ISO感度、ホワイトバランスの設定 | メインシステムからのファイル通信により遠隔設定可能 |
| フォーカスモード制御 | 手動フォーカス(MF)、AF-S、AF-Cなど9つのフォーカスモードの切り替え | 起動時はMFモードで初期化、CASからの指示で変更可能 |
| 画像撮影制御 | 制御ファイルからの撮影コマンド受信 | 画像サイズ、圧縮形式を指定可能 |
| バックグラウンド動作 | タスクトレイアイコンとしてシステムトレイに常駐 | メインウィンドウを非表示化し、システムリソースを最小化 |

### 2-2. システム化対象領域（適用範囲）と影響範囲

#### 適用範囲

- **カメラ制御機能**：カメラとの通信・制御
- **パラメータ管理**：撮影設定（F値、シャッター速度、ISO感度、ホワイトバランス）の管理
- **ファイルベース通信**：CASメインシステムとの制御ファイルによるデータ交換（ポーリング方式）
- **UI機能**：タスクトレイアイコン、コンテキストメニュー
- **設定管理**：XML形式の設定ファイル読み込み・保存

#### 影響を受ける周辺システム

| システム名 | 影響内容 |
|-----------|---------|
| CAS Main System | 制御ファイル作成/更新、結果ファイルの読み取り |
| CameraControl DLL | カメラ制御の具体的な実装、ファームウェア更新時の互換性確認 |
| OS (Windows) | ドライバインストール、USB接続管理 |
| 制御ファイル保存先 | CameraControlFileで指定されるパス |

---

### 2-3. ソリューション方針

- **アーキテクチャ**：WPF + ファイルベース通信によるシンプルかつ保守性の高い設計
- **通信方式**：制御ファイルのポーリング方式（タイマーで10ms毎にファイルをチェック）を採用
- **デバイス抽象化**：CameraControl DLLによるカメラ操作の抽象化で、カメラ変更時の影響を最小化
- **バックグラウンド動作**：タスクトレイ常駐により、ユーザーの作業を妨げない
- **設定管理**：XML形式により外部から設定変更を容易に

| 5 | ファイルベース通信 | CASメインシステムとのファイルポーリング制御 | 高 |
| 6 | タスクトレイUI | 常駐アイコン表示及びコンテキストメニュー操作 | 高 |
| 7 | 設定ファイル読み込み | AccSettings.xmlからのカメラ名、初期設定値の読み込み | 高 |
| 8 | エラーハンドリング | カメラ未接続時、ファイルI/Oエラー時の例外処理 | 中 |

| 10 | ライブビュー機能 | LiveViewFlag確認によるライブビュー撮影実行 | 中 |

### 2-5. データ要件

| データ名 | 主要項目 | 関連データ | 備考 |
|----------|----------|-----------|------|
| 撮影設定 | ImageSize, FNumber, Shutter, ISO, WB, CompressionType | 制御ファイル | CASから制御ファイルに記録される |
| 撮影命令 | ShootFlag, Path | 制御ファイル | 撮影実行フラグと保存先パス、タイマーで監視 |
| オートフォーカス命令 | AutoFocusFlag | 制御ファイル | AF実行フラグ、実行後にリセット |
| ライブビュー命令 | LiveViewFlag (0/1/2) | 制御ファイル | LV実行フラグ、複数の状態値を使用 |
| 設定情報 | CameraName, InitialFocusMode, DefaultFNumber | AccSettings.xml | スタートアップ時に読み込み |
| 終了命令 | CloseFlag | 制御ファイル | 終了シグナルフラグ、ポーリングで検出 |

### 2-6. 関連システムインタフェース要件

| 連携先システム | インタフェース種別 | データ内容 | 頻度 |
|--------------|-----------------|-----------|------|
| CAS Main System | ファイルI/O (制御ファイル) | ShootFlag, AutoFocusFlag, LiveViewFlag, CloseFlag等 | 10ms毎ポーリング |
| CameraControl DLL | .NET DLL参照 | メソッド呼び出し（OpenCamera, SetFocusMode等） | アプリ起動時、制御時 |
| AccSettings | ファイルI/O (XML) | カメラ名、初期設定値、制御ファイルパスの読み込み | 起動時、設定変更時 |

| OS (Windows) | System API | タスクトレイ操作、ウィンドウ管理 | 常時 |

### 2-7. 要件定義不要機能

| 機能名 | 不要となる理由 |
|--------|--------------|
| メインウィンドウUI | タスクトレイアイコンで十分な制御が可能、ユーザー操作は最小化 |
| リモートカメラ切り替え | 1台のカメラのみを対象、複数カメラ追加は行わない |
| ネットワーク対応 | ローカルアクセスのみ |

| 制約種別 | 内容 |
|----------|------|
