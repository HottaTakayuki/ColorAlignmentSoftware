## 10. 記入ガイド 差分整理（ColorAlignmentSoftware_Nice）

## 要約

記入ガイドは Nice 実装の追従観点を先に固定するための章として使えます。条件コンパイル、ADCP / SDCP の責務分離、初期値差分、機種別 geometry を外さないことが重要です。

## 差分一覧

| 観点 | 注意点 | 文書影響 |
|------|--------|----------|
| 条件コンパイル | ForNice、BulkCorrectValueWall、Auto_WriteData、Spec_by_Zdistance、MultiController を文書へ反映する | 7章、8章への影響が大きい |
| I/F の混同防止 | ADCP JSON を使う bulk path と SDCP を使う通常 path を混同しない | 7章、8-3 の整理に影響 |
| 初期値 | Nice 条件では 127 初期化がある | 5章、8-3 の前提値に影響 |
| 機種別 geometry | VP15/23/VS25 系では moduleCount と cabinet/module geometry が変わる | 4章、5章、8章に影響 |

## 更新方針

- 上記 4 観点を文書更新時の確認項目として固定する。
