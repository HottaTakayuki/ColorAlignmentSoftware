## 7. 関連システムインタフェース仕様 差分整理（ColorAlignmentSoftware_Nice）

## 要約

7 章は Nice 実装との差が大きい章です。既存文書は SDCP を主軸にしていますが、Nice 実装では ADCP JSON による bulk backup/restore が追加され、Controller 単位の通信構造も明確になっています。

## 差分一覧

| 観点 | 既存文書 | Nice 実装 | 影響 |
|------|----------|-----------|------|
| Controller I/F | IF-GAP-002 は SDCP 中心 | ADCP JSON による Gap correction value all get/set を追加 | IF 一覧の拡張が必要 |
| 接続先 | 単一 Controller に見える | 127.0.0.x を ControllerID ごとに切替 | 多 Controller 前提へ修正が必要 |
| データ形式 | XML / byte 配列中心 | JSON シリアライズ / デシリアライズを使用 | データ項目定義の補強が必要 |

## 更新方針

- IF 一覧に ADCP JSON I/F を追加する。
- インタフェースデータ項目へ CmdGapCorrectionValueAllGet、CmdGapCorrectionValueSet、JSON 文字列を追加する。
- シーケンスに Controller 単位の bulk backup / restore を追加する。
