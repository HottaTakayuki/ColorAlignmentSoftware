## 8. メソッド仕様 差分整理（ColorAlignmentSoftware_Nice）

## 要約

8 章は章構成を流用できますが、中身は 8-2、8-3、8-4 を中心に再整理が必要です。Nice 実装では bulk restore、補正ループ、ADCP 一括書込み、moire 判定、gamma 取得が入り、既存文書との差が最も大きく出ています。

## 差分一覧

| 節 | 既存文書 | Nice 実装 | 影響 |
|----|----------|-----------|------|
| 8-1 | UI 入口中心 | bulk restore 分岐、位置合わせトグル解除差分あり | 中程度の追記が必要 |
| 8-2 | 計測・補正の主経路 | gamma 取得、moire 判定、補正反復を含む | 大幅更新が必要 |
| 8-3 | SDCP 中心の書込み | ADCP JSON による bulk path を含む | 大幅更新が必要 |
| 8-4 | 補助計算中心 | geometry 分岐、gamma、moire が強く関与 | 大幅更新が必要 |
| 8-5〜8-7 | おおむね維持可能 | ADCP / JSON 系の責務整理が必要 | 軽微から中程度の修正が必要 |

## 更新方針

- 8-1 から 8-4 を Nice 実装基準で更新する。
- 8-7 の相互参照対象に ADCPClass を含めるか、8-3 側で Nice 固有 I/F として閉じるかを決める。
