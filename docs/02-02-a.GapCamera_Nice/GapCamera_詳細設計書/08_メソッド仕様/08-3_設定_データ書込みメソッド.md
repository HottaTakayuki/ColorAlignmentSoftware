### 8-3. 設定・データ書込みメソッド 差分整理（ColorAlignmentSoftware_Nice）

## 要約

8-3 は Nice 実装で大きく増強されています。SDCP を使う通常 path に加え、ADCP JSON を使う bulk path があり、補正値の初期化値や値の詰め方にも Nice 固有差分があります。

## 差分一覧

| 観点 | 既存文書 | Nice 実装 | 影響 |
|------|----------|-----------|------|
| backup / restore | SDCP 中心 | backupGapRegAsync / restoreBulkGapRegAsync が ADCP JSON bulk path を持つ | bulk path の追記が必要 |
| 初期化値 | 128 固定前提 | getGapCvCell_InitializedValue は ForNice 条件で 127 | 初期化値説明の修正が必要 |
| restore の役割 | restore 系の差が薄い | restoreGapRegAsync は通常 path、restoreBulkGapRegAsync は Controller 単位一括 path | メソッド責務の明確化が必要 |
| 値の詰め方 | 単純反映に見える | 2 点補正か平均化かで JSON の値構成が変わる | BulkCorrectValueWall 条件の追記が必要 |

## 更新方針

- 8-3 に ADCP 一括取得・一括設定の節を追記する。
- 初期化値の章内記述を Nice 条件込みへ修正する。
