### 8-2. 業務処理メソッド 差分整理（ColorAlignmentSoftware_Nice）

## 要約

8-2 は Nice 実装との差が最も大きい節です。measureGapAsync と adjustGapRegAsync は既存文書の想定より長く、計測前準備、品質判定、補正反復、条件付き自動書込みまで含みます。

## 差分一覧

| メソッド | 既存文書 | Nice 実装 | 影響 |
|----------|----------|-----------|------|
| measureGapAsync | Gap 計測中心 | Controller ごとの gamma 値取得、露出補正、AutoFocus 保存、camera position 再確認を含む | 手順表の拡張が必要 |
| captureGapImages | Gap 画像撮影中心 | moireArea / moireCheck 系撮影も含む | 撮影パイプラインの再整理が必要 |
| adjustGapRegAsync | 単回補正に見える | Before 保存、補正ループ、Result 評価、必要時の自動書込みを含む | 補正フロー全体の見直しが必要 |
| geometry 決定 | 固定的に読める | VP15/23/VS25 系で moduleCount と cabinet/module size が変わる | 前提条件の明記が必要 |

## 更新方針

- measureGapAsync と adjustGapRegAsync の手順表に gamma 取得、work distance 補正、moire 判定、補正反復を追加する。
- 異常系に gamma 取得失敗、camera position inappropriate、moire 検出を追加する。
