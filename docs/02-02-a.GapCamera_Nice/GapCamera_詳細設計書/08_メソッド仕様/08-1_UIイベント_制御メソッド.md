### 8-1. UIイベント・制御メソッド 差分整理（ColorAlignmentSoftware_Nice）

## 要約

UI イベントの入口構造自体は既存文書と大きく変わりませんが、Nice 実装では restore の bulk 分岐、位置合わせトグルの強制解除、ShootCondition の生成差分が入っています。既存文書のままだと、入口処理は正しく見えても実際の分岐条件を追えません。

## 差分一覧

| 対象 | 既存文書 | Nice 実装 | 影響 |
|------|----------|-----------|------|
| btnGapCamRestore_Click | 通常 restore 前提 | BulkSetCorrectValue 条件で restoreBulkGapRegAsync を呼ぶ | 8-1-2 の分岐整理が必要 |
| btnGapCamRestoreBulk_Click | 一括 restore の独立入口 | 実装上は bulk 系文言と進捗表示が通常 restore と近い | 8-1-3 の記載粒度調整が必要 |
| tbtnGapCamSetPos_Click | ShootCondition を設定値から取得 | new ShootCondition(Settings.Ins.GapCam.Setting_A6400/A7) を生成 | 8-1-4 の前提条件修正が必要 |
| 計測・補正・ROM 開始イベント | 開始前チェック中心 | 位置合わせトグル ON の場合は OFF に戻してから本処理へ進む | 前提条件へ「位置合わせ停止済み」を追加する必要がある |
| 完了メッセージ | 単一文言前提 | bulk / non-bulk で文言が分かれる | メッセージ仕様との整合が必要 |

## 更新方針

- 8-1-2、8-1-3 に bulk 分岐と完了メッセージ差分を明記する。
- 8-1-4 に Nice の ShootCondition 生成差分と対象機種差分を追記する。
- btnGapCamMeasStart_Click、btnGapCamAdjStart_Click、btnGapCamRomStart_Click の前提条件へ「位置合わせ停止済み」を追加する。
