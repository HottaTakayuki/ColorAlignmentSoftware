## 1. ビジネス要件

### 1-1. To-Be業務プロセス概要

CameraControllerSharp.dll は、Sony Alpha カメラ制御用のネイティブライブラリ（CameraController.lib / CameraControllerAlpha）を .NET から利用可能にする C++/CLI ブリッジライブラリである。

※ CameraControllerAlpha は CameraControllerSharp.dll の下位で使うネイティブ制御クラス名であり、上位アプリケーション名の AlphaCameraController とは別物である。

CAS 系システムでは、上位の CameraControl から本 DLL を呼び出すことで、以下の機能を統一的に提供する。

- カメラデバイス列挙および接続・切断
- 撮影画像およびライブビュー画像の取得
- 撮影条件（F値、シャッター速度、ISO、WB、画像サイズ、圧縮形式）の取得・設定
- フォーカスモード、AF 実行、AF エリア種別・座標設定

本 DLL を導入することで、上位アプリはカメラ SDK 固有実装から分離され、再利用性と保守性を向上させる。

---

### 1-2. 業務内容、業務特性（ルール、制約）

| 業務名 | 業務内容 | ルール・制約 |
|--------|----------|-------------|
| デバイス管理 | デバイス列挙、デバイス名取得、接続・切断 | 接続対象は列挙結果に存在すること |
| 画像取得 | 静止画・ライブビュー画像の取得 | 取得可否は bool 戻り値で判定 |
| 撮影設定制御 | 画像サイズ、圧縮形式、露出関連値の設定 | 受け付ける値はカメラ側サポート値に依存 |
| フォーカス制御 | フォーカスモード変更、AF 単発実行、AF/MF ホールド制御 | モード値・ボタン状態値は定義済み列挙体を使用 |
| AF エリア制御 | AF エリア種別・座標の取得／設定 | 座標範囲はカメラ仕様に依存 |

- Windows 開発環境（Visual Studio, C++/CLI ビルド可能）
- CameraController.lib および関連ヘッダ
- USB 接続環境

### 1-5. 概要業務フロー

1. 上位アプリが CameraControllerSharp.dll をロードする
2. CCameraController インスタンスを生成する
3. EnumerateDevices / GetDeviceName で対象カメラを特定する
4. ConnectDevice で接続する
5. 撮影・ライブビュー・設定変更 API を実行する
6. 処理終了時に DisconnectDevice を実行する

---

## 2. システム要件（機能要件）

### 2-1. システム全体像

CameraControllerSharp.dll は、マネージド層とネイティブ層の境界に位置する。

- 上位: CameraControl.dll（C#）
- 本要件対象: CameraControllerSharp.dll（C++/CLI）
- 下位: CameraController.lib / CameraControllerAlpha（ネイティブ制御本体）
- デバイス: Sony Alpha カメラ

---

### 2-2. システム化対象領域（適用範囲）と影響範囲

#### 適用範囲

- CCameraController クラスによるカメラ制御 API 提供
- 圧縮形式・画像サイズ・フォーカス関連列挙値の提供
- ネイティブオブジェクトライフサイクル管理（コンストラクタ／デストラクタ／ファイナライザ）

#### 影響を受ける周辺システム

| システム名 | 影響内容 |
|-----------|---------|
| CameraControl.dll | API シグネチャ変更時にコンパイル・実行影響 |
| AlphaCameraController | CameraControl 経由で間接影響 |
| CameraController.lib | 下位 SDK 仕様変更時に DLL 動作が変化 |
| 実機カメラファームウェア | 制御項目の可否や値範囲に影響 |

---

### 2-3. ソリューション方針

- ラッパー方針: 1メソッド 1責務で CameraControllerAlpha 呼び出しを透過的に中継
- 型方針: 必要な制御値を enum で公開し、上位言語から扱いやすくする
- 管理方針: ネイティブポインタを DLL 内部で保持し、破棄処理を明示
- 拡張方針: 既存 API 互換性を優先し、追加は後方互換を維持した形で実施

---

## 3. システム要件（非機能要件）

### 3-1. 移行要件

新規作成済み DLL の継続運用を前提とし、大規模データ移行は不要。

| 移行対象 | 移行方法 | タイミング | 依存関係 |
|----------|----------|-----------|---------|
| DLL 配布物 | 既存配置先への差し替え | リリース時 | CameraControl 側参照更新 |

---

### 3-2. 品質要件

| 品質特性 | 要件内容 | 指標・目標値 |
|----------|----------|------------|
| 信頼性 | 接続・取得・設定 API が安定して成功/失敗を返す | 連続100回操作で異常終了0件 |
| 保守性 | API 名と責務の一貫性を維持する | 既存メソッド互換を維持 |
| 正確性 | 指定値が正しく下位ライブラリへ伝達される | 設定反映不一致率 0% |
| 機能性 | 上位要求（接続、撮影、設定、AF）を満たす | 2-4 全項目実装 |
| 生産性 | 上位アプリ側実装簡素化を支援 | SDK 直接依存コードを DLL 内に集約 |
| 操作性 | 呼び出し手順が単純である | API 呼び出しシーケンス定型化 |
| 経済性 | 既存資産流用により改修コストを抑制 | 大規模再実装を回避 |

---

### 3-3. 性能要件

| 項目 | 要件内容 | 目標値 |
|------|----------|--------|
| 接続応答時間 | ConnectDevice 実行から結果返却 | 3秒以内（通常環境） |
| 設定応答時間 | Set 系 API の結果返却 | 500ms以内（通常環境） |
| 画像取得開始応答 | GetImage/GetLiveImage 呼び出し応答 | 1秒以内（通常環境） |
| 同時接続数 | 制御対象カメラ数 | 1台 |
